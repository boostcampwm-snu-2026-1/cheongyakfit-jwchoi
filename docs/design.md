# UI/UX 디자인 (Design)

청약핏이 **화면에서 어떻게 보이고 동작하는지**를 정의한다. (코드 구조는 [architecture.md](./architecture.md))

> 🚧 **미작성** — 아직 디자인 작업 전. 아래는 채울 자리(뼈대)다. UI는 텍스트보다 목업/스크린샷이 나을 때가 많으니, 필요하면 화면을 같이 그려가며 채운다.

## 1. 화면 목록 / 플로우
> 🚧 로그인 → 프로필 페이지(자격 항목) → 공고 업로드 → 공고 요약 제시 → 결과 화면(① 요약 ② 자격 판정).

## 2. 화면별 레이아웃 / 와이어프레임
> 🚧 미작성.

## 3. 비주얼 스타일
> 🚧 컬러·타이포·간격, 디자인 토큰. (Tailwind + shadcn/ui 기반)

## 4. 컴포넌트 인벤토리
> 🚧 shadcn/ui 컴포넌트 매핑.

## 5. 상태별 UI
> 🚧 로딩 / 에러 / 빈 상태 / "확인필요" 표시 / 파싱 신뢰도 표시.
