# 룰 검증 예시 (대표 공고 × 프로필 → 손 판정)

[eligibility.md](./eligibility.md)의 룰·기준표가 실제로 "사람이 한 자격 판정"을 만들어내는지 손으로 검증한 fixture. **Phase 2 매칭 엔진 TDD의 기대값 시드**로 그대로 옮긴다(프로필 fixture × 공고 fixture → 기대 판정). 값은 [architecture.md §7·§8](../architecture.md) 스키마를 따른다.

> 모든 수치는 검증용 가상값. 판정 status는 **가능 / 불가능 / 확인필요** 3종, 각 근거·서류는 [eligibility.md](./eligibility.md) 역참조.

## 1. 대표 공고 (notice fixture)

가상 단지 "서울 OO자이" — 서울 소재(10·15 대책으로 **투기과열지구**), **분양가상한제** 적용.

| 필드 | 값 |
|------|-----|
| `announcementDate` | 2026-07-01 |
| `regulationZone` | 투기과열지구 |
| `priceCapApplied` | true |
| `eligibleRegions` | 서울특별시(해당지역 우선) + 수도권 |

**주택형** (`unitTypes[]`) — 판정은 **59㎡** 기준(6종 모두 공급 있음):

| 전용면적 | 분양가 | 일반(가점/추첨) | 신혼 | 생애최초 | 다자녀 | 노부모 | 신생아 |
|---------|-------|---------------|------|---------|-------|-------|-------|
| 59㎡ | 9.8억 | 8 / 5 | 7 | 4 | 2 | 1 | 3 |
| 84㎡ | 13.5억 | 12 / 6 | 5 | 3 | 2 | 1 | 3 |

(59·84 모두 85㎡ 이하 → 신혼·생애최초·신생아 공급 가능. 59㎡ 가점/추첨 비율 = **투기과열 60㎡이하 40%/60%**.)

## 2. 대표 프로필 (profile fixture)

"서울 거주, 혼인 4년차 맞벌이, 자녀 1명(만 1세)" — 무주택 세대주.

| 군 | 필드 | 값 |
|----|------|-----|
| 기본·세대 | `birthDate` | 1992-03-01 (만 34세) |
| | `isHouseholdHead` | true |
| | `residenceSido` | 서울특별시 |
| | `residenceSince` | 2019-01-01 (거주 7년+) |
| 세대원 | `household` | 배우자(1993-05, 기혼, 무주택) + 자녀(2024-09-01 출생, 무주택) |
| 혼인·자녀 | `maritalStatus` | 기혼 |
| | `marriageDate` | 2022-05-01 (혼인 4년차) |
| | `isDualIncome` | true |
| | `children` | [{ status: 출생, birthDate: 2024-09-01 }] |
| 소득·자산 | `householdSize` | 3 |
| | `applicantIncome` | 4,500,000 / 월 |
| | `spouseIncome` | 4,000,000 / 월 (합 8,500,000) |
| | `realEstateAsset` | null |
| | `incomeTaxPaidYears` | 8 |
| 청약통장 | `hasAccount` | true |
| | `accountOpenDate` | 2018-01-01 (가입 8년+) |
| | `depositAmount` | 3,000,000 |
| 주택·이력 | `homelessSince` | 2016-03-01 |
| | `everOwnedHome` | false |
| | `pastWin` | null |
| | `usedSpecialSupply` | false |

**CORE 산출 파생값** (저장 X, 계산):
- 무주택기간 = max(만30세 2022-03-01, 혼인 2022-05-01)~공고일 = **4~5년 구간**
- 부양가족수 = 배우자1 + 미혼 미성년 자녀1 = **2명**
- 통장 가입기간 = 2018-01 ~ 2026-07 = **8~9년 구간**
- 미성년 자녀수 = 1, 영유아(만6세 미만) = 1, 2세 미만 = **있음**
- 가구 월소득 = 8,500,000 (3인 100% = 8,168,429 — [§2.1](./eligibility.md))

## 4. "확인필요" 예시

같은 프로필에서 **`spouseIncome` 미입력**(맞벌이=true인데 배우자 소득 공란)이면:

| 청약유형 | 판정 | 처리 |
|---------|------|------|
| 신혼·생애최초·신생아 | ⚠️ 확인필요 | 가구 소득 합산 불가 → 소득요건 단정 금지. **배우자 월소득만 되묻는다**(미흡분만 — [decisions.md](../decisions.md) D10) |
| 일반·다자녀·노부모 | (위 §3과 동일) | 소득과 무관하거나 이미 불가능 확정 |

→ 매칭 엔진은 입력이 비면 가능/불가능을 단정하지 않고 **확인필요 + 되묻기**로 처리한다([architecture.md §9](../architecture.md)).
